四种通信模式简介

模式 全称 典型引擎 协议 特点
D2D Device to Device COMM_ENGINE_AICPU_TS (AICPU时间片) COMM_PROTOCOL_UBC_CTP Device间直接传输，最高性能
H2D Host to Device COMM_ENGINE_CPU → COMM_ENGINE_AICPU_TS COMM_PROTOCOL_UBC_CTP Host内存写入Device内存
D2H Device to Host COMM_ENGINE_AICPU_TS → COMM_ENGINE_CPU COMM_PROTOCOL_UBC_CTP Device内存写入Host内存
H2H Host to Host COMM_ENGINE_CPU (CPU) COMM_PROTOCOL_UBC_CTP Host间直接传输

typedef struct urma_jfs_wr {
    urma_wr_opcode_t opcode;      // 操作码: WRITE/READ/SEND
    urma_jfs_sge_t *sgl;          // SGE (Scatter-Gather Element) 链表
    uint32_t sge_count;           // SGE 数量
    uint64_t wr_id;               // WR 唯一标识
    urma_jfs_wr_flag_t flags;     // 标志位: SIGNALED/INLINE等
    urma_ordering_t ordering;     // 序保障类型
    union {
        struct {
            urma_target_seg_t *dst_seg;  // 目标segment handle
            uint64_t dst_addr;           // 目标内存地址
            uint32_t len;                // 传输长度 (字节)
        } write;
        struct {
            urma_target_seg_t *src_seg;  // 源segment handle
            uint64_t src_addr;           // 源内存地址
            uint32_t len;                // 传输长度 (字节)
        } read;
        struct {
            uint32_t inline_len;         // inline数据长度
            uint8_t *inline_data;        // inline数据指针
        } send;
    } op;
} urma_jfs_wr_t;

总结

关键共同点

  • 协议统一: 四种模式均使用 COMM_PROTOCOL_UBC_CTP (华为统一总线CTP协议)
  • 三层架构: HIXL → HCOMM → URMA 层次完全一致
  • 底层URMA统一: 最终都是通过 urma_post_jetty_send_wr() → UDMA驱动
  • 门铃机制: 均使用 DSQE优化路径 或 MMIO PI写入 通知硬件
  • 分片大小: kMaxKernelBatchSize = 128
  • API接口: 均遵循 hixl/include/cs/hixl_cs.h 定义的接口规范

关键差异点

  • 执行位置: D2D/H2D/D2H 在 Device Kernel执行，H2H 在 Host CPU直接执行
  • Channel类型: AicpuTsUrmaChannel vs HostCpuUrmaChannel
  • 内存类型: COMM_MEM_TYPE_DEVICE vs COMM_MEM_TYPE_HOST
  • 完成检测: Device路径通过 host_flag，Host路径通过 flag_queue
  • 资源管理: Device路径需要 TransferPool，Host路径需要 FlagQueue
  • 重试机制: Host路径有20分钟超时重试，Device路径无
